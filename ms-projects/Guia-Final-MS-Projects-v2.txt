GUIA TECNICA COMPLETA Y DETALLADA: MICROSERVICIO DE PROYECTOS (MS-PROJECTS)
========================================================================

Este documento contiene la bitacora exacta de implementacion, comandos de PowerShell,
versiones de librerias y resolucion de errores para el microservicio de proyectos.

1. RESUMEN DE VERSIONES Y ENTORNO
--------------------------------
- Node.js: v18+ o v20+
- TypeScript: v5.x o v6.x
- Prisma: v5.x+
- PostgreSQL (Docker): Image postgres:15
- Puerto MS: 3002
- Puerto DB (Externo): 5434 (o 5435 segun configuracion)

2. PASO A PASO: ORDEN DE EJECUCION
----------------------------------

PASO 1: Inicializacion de Carpeta y Node.js
   PS> mkdir ms-projects
   PS> cd ms-projects
   PS> npm init -y

PASO 2: Instalacion de Dependencias
   # Produccion
   PS> npm install express @prisma/client dotenv swagger-ui-express swagger-jsdoc
   # Desarrollo
   PS> npm install -D typescript ts-node-dev @types/express @types/node @types/swagger-ui-express @types/swagger-jsdoc prisma

PASO 3: Configuracion de Infraestructura (Docker)
   # Crear archivo docker-compose.yml
   PS> New-Item -Path . -Name "docker-compose.yml" -ItemType "file"
   # Contenido clave: ports ["5434:5432"]

PASO 4: Variables de Entorno (.env)
   # Crear archivo .env
   PS> New-Item -Path . -Name ".env" -ItemType "file"
   # Contenido clave: DATABASE_URL="postgresql://admin:projectpassword@localhost:5434/ms_projects_db?schema=public"

PASO 5: Modelado y Persistencia (Prisma)
   PS> npx prisma init
   # Definir el modelo Project en prisma/schema.prisma
   PS> npx prisma migrate dev --name init
   PS> npx prisma generate

PASO 6: Configuracion de TypeScript (tsconfig.json)
   PS> npx tsc --init
   # Ajustes obligatorios para evitar error TS5011:
   # "rootDir": "./src"
   # "outDir": "./dist"
   # "include": ["src/**/*"]

PASO 7: Arquitectura de Capas (Estructura de Carpetas)
   PS> mkdir src, src/controllers, src/services, src/factories
   PS> New-Item src/app.ts, src/swagger.ts, src/controllers/project.controller.ts, src/services/project.service.ts, src/factories/project.factory.ts

PASO 8: Script de Desarrollo
   # En package.json agregar:
   # "dev": "ts-node-dev --respawn --transpile-only src/app.ts"

3. COMANDOS DE MANTENIMIENTO (POWERSHELL)
-----------------------------------------
- Levantar Base de Datos:
  PS> docker-compose up -d

- Reinicio Limpio (Si falla la autenticacion P1000):
  PS> docker-compose down -v
  PS> docker-compose up -d

- Correr el Servidor:
  PS> npm run dev

- Ver Datos en el Navegador:
  PS> npx prisma studio

4. BITACORA DE ERRORES SOLUCIONADOS
-----------------------------------
- P1001 (Can't reach DB): El puerto en .env no coincidia con el de docker-compose.yml.
- P1000 (Auth failed): Credenciales antiguas en el volumen de Docker; solucionado con 'down -v'.
- TS5011: TypeScript no reconocia la carpeta raiz; solucionado con 'rootDir' en tsconfig.json.
- Swagger Vacio: Errores de indentacion en comentarios @openapi o rutas mal definidas en swagger.ts.
- Module Not Found: Errores de rutas relativas en los imports de src/app.ts.

5. VALIDACION FINAL
-------------------
1. Acceder a http://localhost:3002/api-docs
2. Ejecutar POST /projects (Debe devolver 201 Created).
3. Ejecutar GET /projects (Debe devolver el JSON del proyecto creado).
