1. DESCRIPCIÓN DEL STACK TECNOLÓGICO
------------------------------------
Para el desarrollo de la plataforma Innovatech, se ha seleccionado un conjunto 
de herramientas modernas orientadas a la escalabilidad, el tipado fuerte y 
la eficiencia en entornos de microservicios.

A. LENGUAJE Y RUNTIME: Node.js con TypeScript
   - Node.js: Entorno de ejecución para JavaScript en el servidor, conocido 
     por su modelo de I/O no bloqueante.
   - TypeScript: Superconjunto de JavaScript que añade tipado estático. 
     Garantiza que los errores de datos se detecten en desarrollo y no 
     en producción (Type Safety).

B. FRAMEWORK WEB: Express.js
   - Framework minimalista para Node.js. Facilita la creación de rutas, 
     manejo de middlewares y gestión de peticiones/respuestas HTTP.

C. PERSISTENCIA Y ORM: PostgreSQL + Prisma v5
   - PostgreSQL: Base de datos relacional (RDBMS) robusta y de código abierto.
   - Prisma: Object-Relational Mapper (ORM) de última generación. Actúa como 
     puente entre el código (objetos) y la base de datos (tablas). Genera 
     clientes de base de datos con autocompletado y validación de tipos.

D. INFRAESTRUCTURA Y DESPLIEGUE: Docker
   - Permite "contenedorizar" la base de datos y la aplicación. Asegura que el 
     entorno de ejecución sea idéntico en la computadora del desarrollador y 
     en los servidores de producción.

E. DOCUMENTACIÓN: Swagger (OpenAPI 3.0)
   - Proporciona una interfaz visual interactiva para que otros desarrolladores 
     puedan probar los microservicios sin necesidad de herramientas externas.

2. ARQUITECTURA DEL SISTEMA (CAPAS)
-----------------------------------
El sistema sigue una arquitectura de capas desacopladas:
- Capa de Rutas/Controlador: Gestiona la comunicación externa (HTTP).
- Capa de Servicio: Donde reside la "Lógica de Negocio" (reglas y cálculos).
- Capa de Modelo/Persistencia: Prisma gestiona el acceso a los datos.

3. GLOSARIO DE CONCEPTOS CLAVE (PARA DEFENSA TÉCNICA)
----------------------------------------------------
- MICROSERVICIO: Componente de software independiente que se encarga de un 
  dominio específico (ej: Tareas o Proyectos).
- ENDPOINT: Punto de acceso o URL de una API donde se envía una petición.
- PAYLOAD: Los datos reales que se envían en el cuerpo de una petición JSON.
- MIGRACIÓN (PRISMA): Proceso de traducir los cambios del modelo de datos 
  en el código a cambios estructurales en la base de datos SQL.
- MIDDLEWARE: Código que se ejecuta entre la recepción de la petición y 
  la respuesta final (útil para seguridad y validaciones).
- CRUD: Acrónimo de Create, Read, Update, Delete (Crear, Leer, Actualizar, Borrar).

4. JUSTIFICACIÓN TÉCNICA (PARA INFORME EFT/PARCIAL)
--------------------------------------------------
"La elección de TypeScript y Prisma responde a la necesidad de Innovatech de 
contar con un sistema mantenible y con baja tasa de errores. La arquitectura 
de microservicios sobre Docker garantiza que la solución pueda escalar 
horizontalmente, permitiendo que cada módulo crezca de forma independiente 
según la demanda del negocio."
